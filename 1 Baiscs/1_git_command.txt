1️⃣ git init
    Converts a normal folder into a Git tracked project
    Creates a hidden .git folder where Git stores history, commits, branches


2️⃣ git status
    Shows:
        Modified files
        New untracked files
        Files ready to be committed    


3️⃣ git add <file>
        Moves files to the staging area
        You write multiple files but clip only some files to submit to commits
        Other files are ignored for now
    

4️⃣ git add .
    Stages:
        All new files
        All modified files
        All deleted files
    in the current directory and subfolders
Always run git status before and after git add .



5️⃣ git commit -m "message"
    Save changes with a message on version control
    Now Git has:
        Who changed
        What changed
        When it changed
        Why it changed


6️⃣ git log
    See commit history
    Shows:
        All commits
        Commit IDs
        Author
        Date
        Message

7️⃣ git log --oneline
    Clean short history
    Shows one commit per line
    Short commit ID + message
    Very clean and readable
