1 + 2. CSS Syntax & 3 Ways to Write CSS

The basic structure of a CSS rule: selector { property: value; }
Read as: "Target this element and apply these styles to it."
/* METHOD 1: Inline CSS — written directly on the element */ <h1 style="color: red; font-size: 40px;">Hello</h1> /* METHOD 2: Internal CSS — inside a <style> tag in <head> */ <head> <style> h1 { color: red; font-size: 40px; } </style> </head> /* METHOD 3: External CSS — link a .css file (INDUSTRY STANDARD) */ <head> <link rel="stylesheet" href="style.css"> </head> /* In style.css file: */ h1 { color: red; font-size: 40px; }

3. Selectors — Targeting HTML Elements

Type Element Selector

/* Targets ALL <p> elements on the page */ p { color: #333; margin-bottom: 14px; }
Result — all paragraphs on this page use the p selector:

Class Class Selector ( .classname )

/* In CSS — the dot means "class" */ .highlight-box { background-color: #fff8e1; border-left: 4px solid #f39c12; padding: 14px 18px; } /* Multiple classes on one element */ <p class="bold-text red-text large">...</p>
Result — .highlight-box class applied:
This div has class="highlight-box" — it gets the yellow left-bordered style.

ID ID Selector ( #idname )

/* In CSS — the hash # means "id" */ #main-title { text-align: center; padding: 20px 0; } /* In HTML */ <h1 id="main-title">CSS File 1 — Foundations</h1>
The centered heading at the top of this page uses #main-title. Each id can only appear once per page.

Group Grouping Selector

/* Apply the same style to multiple selectors at once */ h1, h2, h3 { font-family: 'Segoe UI', sans-serif; } /* Without grouping — repetitive (BAD): */ h1 { font-family: 'Segoe UI', sans-serif; } h2 { font-family: 'Segoe UI', sans-serif; } h3 { font-family: 'Segoe UI', sans-serif; }

Descend Descendant Selector

/* Only targets <p> that is INSIDE a .demo-card */ .demo-card p { color: #555; font-style: italic; } /* Other <p> tags outside .demo-card are NOT affected */
Result — italic grey only on paragraphs inside .demo-card:
I'm a .demo-card div

This paragraph IS inside .demo-card — italic grey ✅

This paragraph is OUTSIDE .demo-card — normal style ✅

Child Child Selector ( > )

/* Only targets DIRECT <p> children — not grandchildren */ .demo-parent > p { color: #e94560; font-weight: bold; } /* The <p> inside a nested div is NOT a direct child → not affected */
Result — red bold only on direct child p:

I am a DIRECT child of .demo-parent → red + bold ✅

I am a GRANDCHILD (inside a nested div) → NOT styled ✅

4 + 5. Cascade & Specificity

The Cascade: When two rules conflict, the one with higher specificity wins. If specificity is equal, the last rule written wins.
/* SPECIFICITY SCORE CHART: Selector Example Score (id, class, element) ──────────────────────────────────────────────────────────── Inline style style="..." 1,0,0,0 ← HIGHEST ID #title 0,1,0,0 Class .box 0,0,1,0 Pseudo-class :hover 0,0,1,0 Element h1 0,0,0,1 Universal * 0,0,0,0 ← LOWEST */ /* Example — which color wins? */ h1 { color: blue; } /* score: 0,0,0,1 */ .heading { color: red; } /* score: 0,0,1,0 → WINS */ #main-title { color: green; } /* score: 0,1,0,0 → WINS over both */ /* Inline style would beat even the #id selector */ <h1 style="color: orange;"> ← this wins over everything
Live specificity demo — same element, 3 selectors competing:

The #main-title heading at the top is navy because the id selector (score 0,1,0,0) beats any element or class selector.

This paragraph has inline style red, BUT also color: green !important — !important wins over everything, even inline.

9. CSS Units — px, %, em, rem

px → absolute pixels, no scaling % → relative to parent element em → relative to current element's font-size (COMPOUNDS when nested) rem → relative to root <html> font-size (DOES NOT compound — preferred)
Live unit demo — all set to "1.25x base size":
Parent container: font-size: 16px
font-size: 20px → always exactly 20px (absolute)
font-size: 1.25rem → 1.25 × 16px (root) = 20px (predictable)
font-size: 1.25em → 1.25 × 16px (parent) = 20px (here same, but compounds if nested!)
width: 60% → 60% of parent width (resize window to see it adapt)
Interview Answer — em vs rem:
em is relative to the current element's font-size. It compounds when elements are nested — child's em multiplies parent's em size, causing unpredictable large sizes deep in the tree.

rem is relative to the root (<html>) font-size only. It never compounds. 1rem = 16px everywhere on the page, regardless of nesting. This makes rem predictable and the preferred unit for font-size and spacing in modern CSS.

10. CSS Variables (Custom Properties)

/* DECLARE variables on :root for global access */ :root { --primary: #0f3460; --accent: #e94560; --success: #27ae60; --radius: 8px; --space-md: 16px; --shadow: 0 2px 12px rgba(0,0,0,0.08); } /* USE variables with var() function */ .card { background: white; border-radius: var(--radius); /* 8px */ padding: var(--space-md); /* 16px */ box-shadow: var(--shadow); border-left: 4px solid var(--accent); /* #e94560 */ } /* Fallback value if variable not defined */ color: var(--primary, blue); /* uses blue if --primary doesn't exist */ /* Dark mode with variables — just swap values! */ .dark-mode { --primary: #ffffff; --accent: #ff6b6b; }

⚡ Quick Revision — Interview Cheat Sheet

/* 3 WAYS TO WRITE CSS */ Inline → style="" highest specificity, avoid Internal → <style> in head good for single pages External → <link> to .css ALWAYS use in real projects /* SELECTORS */ element → p { } all paragraphs .class → .box { } elements with class="box" #id → #title { } element with id="title" * → * { } ALL elements group → h1, h2, h3 { } multiple selectors descendant → .card p { } p inside .card (any depth) child → .card > p { } p directly inside .card only /* SPECIFICITY SCORE — higher wins */ Inline style → 1,0,0,0 #id → 0,1,0,0 .class → 0,0,1,0 element tag → 0,0,0,1 !important → overrides EVERYTHING (avoid) /* COLOR FORMATS */ named → color: tomato; hex → color: #e94560; rgb → color: rgb(233, 69, 96); rgba → color: rgba(233, 69, 96, 0.5); hsl → color: hsl(349, 81%, 59%); hsla → color: hsla(349, 81%, 59%, 0.5); /* BACKGROUNDS */ background-color: #0f3460; background-image: linear-gradient(to right, #0f3460, #e94560); background-image: radial-gradient(circle, #f39c12, #e74c3c); background-image: url("img.jpg"); background-size: cover | contain | auto; background-position: center | top | 50% 50%; background-repeat: no-repeat | repeat | repeat-x; /* TYPOGRAPHY */ font-family: 'Segoe UI', Arial, sans-serif; font-size: 1rem; /* base = 16px */ font-weight: 400 | 700 | 300; /* normal, bold, light */ font-style: normal | italic; line-height: 1.6; /* unitless — recommended */ text-align: left | center | right | justify; text-decoration: none | underline | line-through; text-transform: uppercase | lowercase | capitalize; letter-spacing: 2px; text-shadow: 2px 2px 4px rgba(0,0,0,0.3); /* Shorthand: */ font: italic 700 1.2rem/1.6 'Segoe UI', sans-serif; /* UNITS */ px → absolute, exact, doesn't scale % → relative to parent em → relative to current font-size, COMPOUNDS in nesting rem → relative to root html font-size, DOES NOT compound ✅ vw → 1% of viewport width vh → 1% of viewport height /* CSS VARIABLES */ :root { --primary: #0f3460; } /* declare */ color: var(--primary); /* use */ color: var(--primary, blue); /* with fallback */ /* KEY INTERVIEW QUESTIONS */ Q: Difference between em and rem? A: em is relative to current/parent font-size, compounds in nesting. rem is relative to root html font-size, never compounds. Use rem. Q: What is specificity? A: Score system deciding which CSS rule wins when rules conflict. inline > #id > .class > element Q: What is the cascade? A: Rules flow top to bottom. If specificity is equal, last rule wins. Q: background-size cover vs contain? A: cover → scales to fill element entirely (may crop image) contain → scales to fit entire image (may leave empty space) Q: What are CSS variables? A: Custom properties declared with --name on :root, used with var(). Dynamic — can be changed by JavaScript at runtime.
