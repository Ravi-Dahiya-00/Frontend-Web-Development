CSS File 2 — Box Model, Display & Position

The most asked CSS interview topics — all with live examples.

1. The Box Model

Every HTML element is a rectangular box made of 4 layers — from inside out: Content → Padding → Border → Margin.

/* The 4 layers of every element */ .box { width: 300px; /* content width */ height: 120px; /* content height */ padding: 20px; /* space INSIDE border (takes background color) */ border: 6px solid #0f3460; /* the visible line */ margin: 20px; /* space OUTSIDE border (always transparent) */ }
Live Box Model — hover in DevTools to see each layer:
MARGIN (transparent space — pushes other elements away)
BORDER (the visible line)
PADDING (space inside border — shares background color)
CONTENT (actual text / image)

Key difference: Padding = inside (pushes content from border). Margin = outside (pushes elements from each other). Margin is always transparent.

DevTools Tip: Right-click any element → Inspect → scroll to the bottom of the Styles panel. You'll see the exact Box Model diagram with live values for that element.

2. box-sizing: border-box

The single most important CSS trick. Fixes the biggest source of confusion in the Box Model.

/* DEFAULT (content-box) — confusing: */ .box { width: 200px; /* content only */ padding: 20px; /* adds 20+20 = 40px */ border: 5px; /* adds 5+5 = 10px */ } /* ACTUAL size = 200 + 40 + 10 = 250px ← NOT 200px! */ /* BORDER-BOX — intuitive: */ * { box-sizing: border-box; } /* ← put this in every project's reset */ .box { width: 200px; /* TOTAL width including padding and border */ padding: 20px; /* fits INSIDE the 200px */ border: 5px; /* fits INSIDE the 200px */ } /* ACTUAL size = exactly 200px ✅ */
Live demo — both divs have width: 200px, padding: 20px, border: 5px:

actual = 250px ❌

Takes more space than expected

border-box ✅
width: 200px set

Rule: Always start every CSS file with * { box-sizing: border-box; }. Every professional project does this.

width: 300px; /* fixed — always 300px */ width: 60%; /* relative to parent — responsive */ width: 100%; /* full width of parent */ max-width: 400px; /* never wider than 400px even if % wants more */ min-width: 200px; /* never narrower than 200px */ /* Best practice for page containers: */ .container { width: 100%; max-width: 1200px; margin: 0 auto; /* centers horizontally */ }

4. Margin, Padding & Border

Margin — space OUTSIDE the element

/* Shorthand patterns — memorise these */ margin: 20px; /* all 4 sides = 20px */ margin: 20px 40px; /* top+bottom=20px, left+right=40px */ margin: 10px 20px 30px 40px; /* top, right, bottom, left (clockwise) */ margin: 0 auto; /* centers block element horizontally */ /* Individual sides */ margin-top: 10px; margin-right: 20px; margin-bottom: 10px; margin-left: 20px; /* MARGIN COLLAPSE — vertical margins don't add up, larger one wins */ .box-a { margin-bottom: 30px; } .box-b { margin-top: 20px; } /* Gap between them = 30px (NOT 50px) */
Live margin demo:

Padding — space INSIDE the element

/* Same shorthand pattern as margin */ padding: 20px; /* all 4 sides */ padding: 10px 20px; /* top+bottom, left+right */ padding: 10px 20px 30px 40px; /* top, right, bottom, left */ padding-top / padding-right / padding-bottom / padding-left

6. Position

position: static; /* DEFAULT — normal flow, top/left/etc have no effect */ position: relative; /* nudge from normal position, space still reserved */ position: absolute; /* remove from flow, position inside nearest relative parent */ position: fixed; /* remove from flow, position relative to browser window */ position: sticky; /* relative until scroll threshold, then fixed */

position: relative — nudge from normal spot

.box { position: relative; top: 20px; /* moves 20px DOWN from where it would normally be */ left: 10px; /* moves 10px RIGHT from where it would normally be */ } /* Original space is still RESERVED — other elements don't fill it */

position: absolute — place anywhere inside a parent

.parent { position: relative; /* ← REQUIRED: makes parent the reference point */ } .child { position: absolute; top: 10px; right: 10px; /* places child 10px from top-right of parent */ } /* Perfect center trick with absolute: */ .child { position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%); }

The Golden Rule of absolute positioning: The parent must have position: relative. Without it, the child positions itself relative to the entire page — not the parent.

position: fixed — sticks to the browser window

/* Stays on screen even when you scroll */ .navbar { position: fixed; top: 0; left: 0; right: 0; /* stretches full width */ z-index: 100; /* stays above other content */ } .back-to-top { position: fixed; bottom: 20px; right: 20px; }

7. Z-index — Controlling Layers

When elements overlap, z-index decides which one appears on top. Higher = in front. Only works on positioned elements (not static).

/* Higher number = appears in FRONT */ .modal { position: fixed; z-index: 999; } /* topmost — modal/popup */ .navbar { position: fixed; z-index: 100; } /* above content */ .dropdown{ position: absolute; z-index: 50; } /* above nearby content */ .card { position: relative; z-index: 1; } /* slightly raised */ /* z-index: -1 → goes BEHIND normal elements */ .background-shape { position: absolute; z-index: -1; }

⚡ Quick Revision — Interview Cheat Sheet

/* BOX MODEL — 4 layers inside to outside */ Content → Padding → Border → Margin content = actual text/image padding = space inside (takes background color) border = visible line margin = space outside (always transparent) /* BOX-SIZING */ * { box-sizing: border-box; } /* always add to reset */ content-box → width = content only (default, confusing) border-box → width = content + padding + border (use this ✅) /* WIDTH */ width: 300px; /* fixed */ width: 60%; /* relative to parent */ max-width: 1200px; /* never wider than this */ width: 100%; max-width: 1200px; margin: 0 auto; /* centered container */ /* MARGIN SHORTHAND */ margin: 20px; /* all sides */ margin: 20px 40px; /* top+bottom, left+right */ margin: 10px 20px 30px 40px; /* top right bottom left (clockwise) */ margin: 0 auto; /* center a block element */ /* MARGIN COLLAPSE */ Two vertical margins meeting → larger one wins (they don't add up) Horizontal margins never collapse. /* PADDING SHORTHAND — same pattern as margin */ padding: 20px; padding: 10px 20px; padding: 10px 20px 30px 40px; /* BORDER */ border: 2px solid #color; border-style: solid | dashed | dotted | double | none; border-radius: 50%; /* circle */ border-radius: 8px; /* rounded corners */ /* DISPLAY */ block → new line, full width, width/height works inline → same line, content width only, width/height ignored inline-block → same line, width/height WORKS (best of both) none → completely hidden, takes NO space display:none vs visibility:hidden: none → element gone, no space taken hidden → element invisible, space STILL taken /* POSITION */ static → default, top/left etc ignored relative → nudge from normal spot, space still reserved absolute → removed from flow, position inside nearest relative parent fixed → removed from flow, sticks to browser window on scroll sticky → relative until scroll threshold, then sticks /* THE GOLDEN RULE OF ABSOLUTE POSITIONING */ Parent must have position: relative Without it → child positions relative to the whole page /* PERFECT CENTER TRICK */ .child { position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%); } /* Z-INDEX */ Only works on positioned elements (not static) Higher number = appears in front z-index: 999 → modals/popups z-index: 100 → navbar z-index: -1 → behind everything /* KEY INTERVIEW QUESTIONS */ Q: Difference between margin and padding? A: Padding is space inside the element (between content and border). Margin is space outside the element (between elements). Margin is always transparent. Padding takes the background color. Q: What is box-sizing: border-box? A: Makes width/height include padding and border, not just content. Prevents elements from being larger than their set width. Q: Difference between display:none and visibility:hidden? A: display:none removes the element completely — no space taken. visibility:hidden hides it but the space remains. Q: Difference between position absolute and fixed? A: absolute → positioned relative to nearest positioned ancestor. fixed → positioned relative to the browser window, stays on scroll. Q: What is position sticky? A: Behaves like relative until it hits a scroll threshold (like top:0), then sticks like fixed until its parent scrolls away.
